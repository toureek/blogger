##### 好的计算机算法（标准模块化组件 对效率的影响）
+ 好的算法衡量标准： 算法复杂度（高德那），去比较大量数据（大量级数据，近乎无限大）的因素。
+ 把主要问题抽象出来，提高效率， 少做无用功。
+ 冒泡排序  插入排序 (n平方)
+ 归并排序  (n x LogN)
+ 快速排序  (一大堆无序的数字，随机挑选一个数字作为【枢值53】，接下来将所有要排序的数字分为两部分，一部分大于等于【枢值】，一部分小于【枢值】，从上面已经分出的两对数字中，分别再找出一个新的枢值（这两堆中，新的枢值一个肯定大于53， 一个小于53），再次排序，这样原本的数字，已经分为4组了)， 依次向下使用。  复杂的还是 （n x LogN）; 但快速排序在极端情况下的复杂的是（n平方），与冒泡排序一样糟糕。 
+ 排序算法的极限就是 (n x LogN)，所以 没有更快的算法了。
+ 从递推(从正向)到递归（从逆向, 像堆栈这种数据结构） 5! = 5 x 4!;  4! = 4 x 3!;

##### Just Do It.
+ 等到有时间去做，就是永远不会有时间去做。
+ 干一行爱一行的特性，是永远不会被时代淘汰的。
+ 人要走出舒适区 才能发展进步 格局会越来越高 见识越来越深。
+ 乐高积木的模块化，通用化使用，可以映射到工作中时间的模块化和通用化处理。
+ 世界上任何事情 只要到越做越复杂的时候，就是到了应该越做越简单的时候。
+ 很多事情都是从简单到复杂，再重新回归简单。
+ 一个大的问题，如果能拆解为两个等价的简单问题，就很有可能事半功倍。
+ 当一个问题越来越复杂，在解决问题的过程中，复杂度会呈现几何数量的增加。（对不同规模的问题 要有不同的思路）
+ 先学会服从，再获得领导力 + 合作。
+ 用碎片化时间学习和大片时间学习，意义都不大，真正的意义在于学好并用好。
+ 凡事需要就事论事，不以个人喜好主观判断，也不能以以往的经验生搬硬套到新问题上。
+ 少做事 是提高效率的开始。 在一对数字中寻找【中值】
